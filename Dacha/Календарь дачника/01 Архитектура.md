## API (справка, см. `ARCHITECTURE.md` за полным списком)
```
POST /auth/register  POST /auth/login  GET /auth/me
POST /billing/create-payment  POST /billing/webhook
GET /gardens  GET /crops  POST /plantings  POST /actions
GET /today?garden_id=  GET /weather?garden_id=  GET /recommendations?garden_id=
GET /guide  GET /feed
```

## Логика рекомендаций (3 слоя)
1. Культура (`watering_freq_days`, `harvest_days`, `frost_sensitive`)
2. Локация (`climate_zone`, регион)
3. Погода (`temp_min`, `frost_risk`, `heat_risk`)

Теплица (`conditions='greenhouse'`): снимает frost_alert, +30% к интервалу полива.

## Приоритеты задач в `GET /today`
1. `frost_alert`
2. `transplant_due`
3. `watering_due`
4. `harvest_due`
5. `reminder`

## Известные ограничения
- Android unit-тесты на Windows: была проблема с кириллическим путём проекта (`ClassNotFoundException`) — **исправлено 2026-06-25** (build dir перенесён в `%LOCALAPPDATA%`). См. [[08 Статус и бэклог]].
- Open-Meteo иногда недоступен с VPS (таймаут/502).
- RuStore Push требует точного совпадения package name (без `.debug`).
