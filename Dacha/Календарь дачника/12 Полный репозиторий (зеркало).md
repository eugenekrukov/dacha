# Полный репозиторий — зеркало в Obsidian

> обновлено: 2026-06-28, commit `1dc79dd`

С 2026-06-28 часть репозитория видна прямо в Obsidian как **живые ссылки на файловую систему**, а не копии:
- **junction** (`mklink /J`) — для целых папок: вся папка и её содержимое видны как есть, новые файлы внутри появляются автоматически.
- **hardlink** (`mklink /H`) — для одиночных файлов: тот же файл на диске под двумя путями, правки синхронны в обе стороны мгновенно **пока файл правится in-place**.

Использовали junction вместо копирования, чтобы не было двух версий истины — репо остаётся единственным источником.

> ✅ **Обнаружено и окончательно починено 2026-07-02.** Изначально были hardlink-и (`mklink /H`) —
> разорваны были 3 из 11 (`summary.md`, `session-note.md`, `CONVENTIONS-backend.md`, ровно те, что
> правились через git/редакторы недавно). Причина: файл был перезаписан через паттерн
> temp-file+rename (так делают `git pull`/checkout и многие редакторы при «атомарной» записи) —
> rename создаёт НОВый inode по тому же пути, вторая ссылка (в Obsidian) остаётся указывать на
> старый inode со старым содержимым, и никакой ошибки при этом не возникает — расхождение тихое.
>
> **Все 11 ссылок пересозданы как symlink** (не hardlink) — symlink хранит путь, а не inode, и
> переживает rename-перезапись цели, так что повторный разрыв больше не должен происходить.
> Владелец включил Developer Mode (Windows 11: Параметры → Система → Для разработчиков) — без
> него Windows не даёт создавать file-symlink без прав администратора.
>
> ⚠️ **Готча при пересоздании (на будущее, если понадобится ещё раз)**: `New-Item -ItemType
> SymbolicLink` в Windows PowerShell 5.1 **не работает** без реальных прав администратора даже при
> включённом Developer Mode — известное ограничение этой версии PowerShell (не использует API-флаг
> «непривилегированного» создания symlink, которым как раз даёт пользоваться Developer Mode).
> Работает `cmd /c mklink "<путь-в-Obsidian>" "<путь-в-репо>"` (без `/H` и без `/D` — обычный
> file-symlink) — та же команда из `cmd.exe`, а не через `New-Item`.

## Что НЕ подключено (осознанно)
- `android/`, `backend/`, `web/` целиком — это код, а не доки; junction всей папки затащил бы в Obsidian тысячи файлов кода/сборки и сделал бы граф/поиск бесполезными. Если нужен конкретный md-файл оттуда — добавлять точечным hardlink (как `CONVENTIONS.md`).
- `landing/` целиком — там HTML/assets для деплоя, не контент для базы знаний; подключён только `README.md`.
- `design-preview/` — HTML-превью экранов, не md; можно junction'нуть отдельно, если понадобится листать превью в Obsidian.
- `.claude/skills/*/SKILL.md` — это определения инструментов для Claude Code, не знания о продукте; не имеет смысла видеть их рядом с базой знаний проекта.
- `ТЗ.pdf`, картинки, бинарники — Obsidian их откроет как есть внутри `Документация/`, если они лежат в `docs/` (уже подключены через junction), но отдельно не индексировались.

## Если нужно добавить что-то ещё
Для **целой папки только с документами** — junction (как `Документация`). Для **одного файла внутри кодовой папки** — hardlink (как `CONVENTIONS-android.md`). Не копировать содержимое вручную — расхождения неизбежны.
