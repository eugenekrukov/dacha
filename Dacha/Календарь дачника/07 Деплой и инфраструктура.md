## ⚠️ SSH — только из PowerShell
`ssh hetzner` работает **только из PowerShell**, не из Bash-инструмента.
- Ключ `~/.ssh/hetzner` зашифрован, лежит в Windows ssh-agent (именованный канал).
- PowerShell использует Windows OpenSSH → ключ из агента автоматически.
- Bash (POSIX/MSYS) ищет ключ через `SSH_AUTH_SOCK`, агент не запущен → `Permission denied`.

Проверка:
```powershell
ssh-add -l
ssh hetzner 'whoami'   # -> root
```

## ⚠️ Кавычки в SSH-командах из PowerShell
1. Удалённую команду оборачивать **одинарными** кавычками, без `"` внутри.
2. Не использовать `"`, `<`, `>` внутри команды — ломает парсинг.
3. Многострочные скрипты — через here-string в stdin (`bash -s`), не ставить `set -e` первой строкой (PowerShell может добавить BOM).

## Деплой backend
```powershell
# локально: влить в main и запушить
git checkout main; git merge --ff-only <branch>; git push origin main
```
```bash
# на VPS
cd /var/www/dacha-api && git fetch origin && git reset --hard origin/main
cd backend && npm install   # если менялись зависимости
pm2 restart dacha-api
```

## Деплой web
```bash
cd /var/www/dacha-api/web && npm ci && npm run build
rm -rf /var/www/dacha-web/* && cp -r dist/* /var/www/dacha-web/
```
