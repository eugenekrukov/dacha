## Почему не как у Perplexity (по AI)

Perplexity советовал on-device TensorFlow Lite + MobileNetV2, обучить на PlantVillage. Для этого
приложения это плохой фит:
1. **PlantVillage не содержит дефицитов микроэлементов** — главного кейса. Нужен свой размеченный
   датасет (недели работы).
2. **Лабораторные модели разваливаются на реальных фото с грядки** (domain gap, см. PlantDoc).
3. **On-device покрывает только Android**, а есть ещё web-клиент; всё приложение и так серверное.

Когда дойдём до Части 2 — рекомендуемый путь **Claude Vision на сервере** (без обучения и датасета,
видит дефициты+болезни+вредители, один код на web и Android, гейт квотой/paywall). Но это позже.

## Текущее состояние

Болезни/вредители существуют только встроенно в культуру (`crops.diseases` / `crops.pests`, JSONB,
миграция 005) и показываются лишь в карточке культуры. Нет: отдельного просмотра, поиска,
кросс-культурного взгляда, фото, и **вообще нет дефицитов микроэлементов**.

## Модель данных

Сущность — **проблема растения** трёх видов: `deficiency` | `disease` | `pest`.

### Миграция `028_plant_guide.sql`
```sql
guide_entries:
  id, slug (UNIQUE), name, kind, element (для deficiency: K/Ca/Mg/N/Fe/B…),
  category, danger (1..3), description, symptoms, conditions, treatment,
  prevention, season, image_url, image_credit, search_text
  + GIN-индекс по to_tsvector('russian', search_text)

crop_guide_entries (M:N + признаки СПЕЦИФИЧНЫЕ для культуры):
  crop_id, entry_id, signs, image_url, image_credit, PRIMARY KEY(crop_id, entry_id)
```
«Огурец + узкий кончик → недостаток калия» = запись-дефицит `K` ↔ культура «Огурец» с `signs` и фото.

### Миграция `029_seed_plant_guide.sql`
Идемпотентный seed (миграции прогоняются каждый деплой → `IF NOT EXISTS` / `ON CONFLICT`).
Стартовый набор: ключевые дефициты (N, P, K, Ca, Mg, Fe, B), несколько болезней и вредителей,
кросс-ссылки на культуры для хедлайн-примеров. Болезни/вредители позже добутстрапить из
`crops.diseases`/`crops.pests` + `docs/crops-knowledge-base.md` (дедуп по имени).

## API (бэкенд, зеркало `routes/crops.js`)

- `GET /guide?kind=&crop_id=&q=` — публично (как `/crops`), фильтр по виду/культуре + поиск.
- `GET /guide/:slug` — деталь + поражаемые культуры (join) с per-crop `signs`.
- `POST /guide`, `PUT /guide/:id` — admin only (`requireAdmin`).
- Тесты в существующий vitest-сьют.

## Картинки (license-free для РФ)

Источники из чата: CC0 (Pixabay/Pexels/Pixnio) → RuPixel (росс. серверы) → свои/Canva-схемы для
дефицитов, которых не найти. **Каждое изображение — проверять лицензию**; поле `image_credit` для
атрибуции. Раздача — статика nginx (`/var/www/dacha-web/media/guide/…`), как уже раздаётся web.
На старте можно текст+иконки, фото добивать инкрементально.

## Контент-долг

**Фото оставшихся записей справочника (~25 из 68).** На 2026-06-15 наполнено 43/68 фото с
Wikimedia Commons (миграция 034). Остаток — редкие болезни/вредители, которых нет в свободном
доступе на Commons (либо только водяные знаки Bugwood / лабораторные микрофото / двусмысленные
кадры): аскохитоз, фомоз, фомопсис, рамуляриоз, парша обыкновенная, церкоспороз, чёрная ножка,
вертициллёзное увядание, дидимелла, фузариоз донца, шейковая гниль, слизистый бактериоз,
угловатая пятнистость, корневая гниль; вредители — землянич. долгоносик, малинно-земляничный
долгоносик, земляничный клещ, гороховая плодожорка, крыжовниковая огнёвка, луковый
скрытнохоботник, морковная листоблошка, стеблевая нематода, зонтичная тля, бахчевая тля; а также
бор-дефицит.

Тактика дозаполнения: **искать в других свободных источниках** — iNaturalist (research-grade,
часть фото под CC-BY/CC-BY-SA/CC0), GBIF, EPPO Global Database, региональные стоки. Лицензию каждой
картинки проверять построчно (как делали с Commons), атрибуцию класть в `image_credit`. Пайплайн
готов (`guide-photos-tmp/`): достаточно сменить источник на API iNaturalist. Выход ожидается
небольшой — многие виды узкоспециализированы. Альтернатива — свои фото пользователя с грядки.
