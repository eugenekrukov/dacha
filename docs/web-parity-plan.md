# План доработки веб-версии до паритета с Android

> Контекст: веб (`web/`, `https://dacha.studio1008.com/app/`) отстаёт от Android по разделам и
> функциям. Последний коммит по вебу — `78aa43c` (2026-06-21). После него на Android ушёл пласт
> фич, которых на вебе нет. Бэкенд и БД единые — почти всё уже готово на сервере.
> Связанные доки: `docs/web-migration-plan.md` (история миграции), спеки
> `docs/superpowers/specs/2026-06-22-personal-feed-design.md`,
> `docs/superpowers/specs/2026-06-22-unified-journal-entry-design.md`.

## 1. Структурные различия (разделы / навигация)

| | Android | Web |
|---|---|---|
| Нижняя/верхняя навигация | Сегодня · Календарь · Посадки · **Профиль** | Сегодня · Посадки · Календарь · Справочник · Ещё |
| Профиль / «Мой участок» | ✅ вкладка с табами **Лента / Статистика / Справочник** | ❌ нет вообще |
| Онбординг-выбор культур | ✅ `OnboardingCropsScreen` после создания участка | ❌ сразу на «Сегодня» |
| Обучающий оверлей (coach-marks) | ✅ `TutorialIntroScreen` | ❌ нет |
| Аналитика | отдельный `AnalyticsScreen` | слита в `HarvestsScreen` (ок, не пробел) |
| Регистрация | отдельный экран | совмещена с входом (ок, не пробел) |

**Главный структурный пробел — отсутствие раздела «Профиль / Мой участок» с лентой.**

## 2. Функциональные различия (за теми же экранами)

| Фича | Android | Web | Бэкенд |
|---|---|---|---|
| Лента «Мой участок» (хронопоток action+note+photo, photo-only, вехи сезона, группировка по месяцам, пагинация) | ✅ | ❌ | ✅ `GET /feed` готов |
| Вехи сезона (посев / первый урожай / завершение) | ✅ в ленте | ❌ | ✅ (внутри `/feed`) |
| Единый блок «действие + заметка + фото» в Журнале | ✅ одна карточка | ❌ журнал показывает только заметку, без фото | данные есть |
| Фото, привязанное к действию (загрузка) | ✅ | ✅ есть (`ActionLogSheet.uploadPhoto`) | ✅ |
| Замена / удаление фото, удаление записи действия с фото (`3cf0c0c`) | ✅ | ⚠️ проверить/добить | ✅ |
| Скрытие авто-заметок + культура на карточке ленты (`b593157`) | ✅ | ❌ (часть ленты) | ✅ |
| Фото-дневник на карточке посадки | ✅ | ✅ `PhotoDiary` | ✅ |

## 3. Платформенные пробелы (уже зафиксированы в migration-plan)

- **Web Push** — уведомлений нет; канал = email (Brevo). Низкий приоритет.
- **Геолокация** — нет reverse-geocode, кнопки «📍 моё местоположение» нет.
- **Snooze задач** — на вебе только dismiss советов.
- ~~Сброс пароля~~ — ✅ уже есть (`PasswordResetScreen`).

## План доработки (по приоритету ценности)

### P1 — Раздел «Профиль / Мой участок» с лентой  ⭐ ключевой пробел  ✅ СДЕЛАНО + ЗАДЕПЛОЕНО (2026-06-23)
Перенесён Android `ProfileScreen` (`baa79e9`) на веб. Бэкенд `GET /feed?limit&offset` готов.
- [x] `api/client.ts` + `api/types.ts`: метод `getFeed` + `deleteAction`, типы `FeedItem`/`FeedResponse` (`action|photo|milestone`).
- [x] `screens/ProfileScreen.tsx`: табы **Лента / Статистика / Справочник**.
  - Лента: карточки по типам (action/photo/milestone), группировка по месяцам, бесконечная
    подгрузка через `IntersectionObserver` + `next_offset`. Полноэкранный просмотр фото с
    заменой/удалением фото и удалением записи (фолдинг P3). Вехи: посев/первый урожай/завершение.
  - Статистика: хаб-ссылки на `/harvests` и `/journal`.
  - Справочник: хаб-ссылки на `/crops` и `/guide`.
- [x] Навигация (`Layout.tsx` + `App.tsx`): «Справочник» в primary-баре заменён на **«Профиль»**
      (иконка `User`), Справочник уехал в «Ещё» — зеркалит Android-табы. Маршрут `/profile`.
- [x] Карточка ленты: культура на карточке; авто-заметки/пустые отметки фильтрует бэкенд `/feed`.
- [x] `components/EntryCard.tsx` — общая карточка «действие+заметка+фото» (фундамент под P2).
- [x] Проверено в превью (demo@dacha.ru): табы, группировка по месяцам, вехи рендерятся; tsc чист, консоль без ошибок.
- [x] **Задеплоено 2026-06-23** (`e5913fe`): VPS `reset --hard origin/main` → `npm ci && npm run build`
      → `/var/www/dacha-web`; `/app/` отдаёт 200 с новым бандлом, `/feed` живой (401 без токена).

### P2 — Паритет журнала с Android  ✅ СДЕЛАНО (не задеплоено)
> **Уточнение по итогам анализа кода:** единый блок «действие + заметка + **фото**» — это
> **лента** (Android `ActionFeedCard`), а НЕ журнал. Android `JournalEntry` фото не показывает
> (`GET /actions` их не отдаёт) — только иконка + действие + культура + ручная заметка. Фотоблок
> уже закрыт лентой в P1. Реальные пробелы веб-журнала vs Android — фильтр по культуре и удаление.
- [x] **Фильтр по культуре** (чипы, зеркало Android `cropFilter`); чипы только по культурам
      с видимыми (ручными) действиями — без «пустых» чипов.
- [x] **Удаление записи** с диалогом-подтверждением; свёрнутая серия (`collapseActions`) удаляет
      все свои `ids` (`ActionGroup.ids` добавлен в `api/schedule.ts`). Иконка действия в строке.
- [x] Проверено в превью: фильтр (Перец), диалог удаления + отмена, tsc чист, консоль без ошибок.
- [x] **Задеплоено 2026-06-23** вместе с P1 (`e5913fe`).

## Оценка
- P1+P2+P3 — основной объём паритета, ~1.5–2.5 недели соло (логика серверная, фронт зеркалит Android).
- P1 даёт максимум ценности (новый раздел + социализационный «шов»).
- Деплой: пересборка `web/` → `/var/www/dacha-web` (миграций не нужно — `/feed` уже в проде).
