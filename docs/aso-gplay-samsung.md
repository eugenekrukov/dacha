# ASO для Google Play и Samsung Galaxy Store — «Календарь дачника»

> Карточки для **бесплатных ad-supported сборок** (`gplay`/`samsung`). Отличие от RuStore
> (`docs/aso-rustore.md`): там — платная подписка «Дачник Про» через ЮKassa, без рекламы; здесь —
> приложение **полностью бесплатное**, доступ ко всем функциям открыт сразу, монетизация —
> ненавязчивая реклама (Yandex Mobile Ads SDK). Описание скорректировано: убраны фразы про подписку
> и пробный период, добавлен акцент на «бесплатно».

**Пакет**: `ru.dachakalend.app` (отдельные сборки/флейворы `gplay`/`samsung`)

---

## Название (Google Play — лимит 30 символов, Samsung — лимит 50)

```
Календарь дачника: сад и огород
```

30 символов — укладывается в лимит обоих сторов. Если Google Play вдруг обрежет (считает иначе) —
запасной вариант:

```
Календарь дачника: огород
```

### Изменения относительно RuStore-версии
- Убрана фраза «Установите ... — и ваш сад...» как CTA про пробу (там была завязана на «7 дней
  бесплатно»/подписку — здесь это не нужно, всё доступно сразу).
- Добавлены два явных «бесплатно»-маркера: после первого абзаца и перед финальным CTA — отрабатывают
  и ASO (запрос «бесплатно») и ожидания пользователя (реклама не станет сюрпризом).

## Чувствительные разрешения и Data Safety (Google Play) / данные (Samsung)

В отличие от RuStore-декларации (`docs/aso-rustore.md`), для `gplay`/`samsung` сборок:

```
ACCESS_COARSE_LOCATION / ACCESS_FINE_LOCATION — определение населённого пункта участка для
  прогноза погоды и агро-рекомендаций.
POST_NOTIFICATIONS — напоминания о дачных работах и оповещения о заморозках/жаре.
AD_ID (com.google.android.gms.permission.AD_ID, добавляется Yandex Mobile Ads SDK) —
  идентификатор рекламы для показа и измерения эффективности рекламы.
```

### Google Play Data Safety — категории для заполнения

| Категория данных | Собирается? | Передаётся третьим лицам? | Назначение |
|---|---|---|---|
| Местоположение (приблизительное/точное) | Да | Нет | Функциональность приложения (погода, климат-зона) |
| Email-адрес | Да | Нет | Аккаунт/вход |
| ID пользователя | Да | Нет | Функциональность приложения |
| Контент приложения (посадки, действия, урожай) | Да | Нет | Функциональность приложения |
| Push-токен (ID устройства) | Да | Нет | Уведомления |
| Рекламный ID (Advertising ID) | Да | **Да — рекламной сети (Яндекс)** | Реклама/маркетинг |

⚠️ **Отличие от RuStore**: пункт «История покупок (статус подписки)» в `gplay`/`samsung` сборках
**не применим** — биллинг отключён (`PAYMENTS_ENABLED=false`), Paywall не открывается.

> Перед заполнением формы Data Safety в Play Console — свериться с фактическим SDK
> (`mobileads:7.12.0`), т.к. он может собирать дополнительные технические идентификаторы
> (например, через AppMetrica) — проверить в документации Yandex Mobile Ads, что именно отправляется
> при `MobileAds.setUserConsent(true)`.

## Открытый вопрос — согласие на рекламу

В коде (`Ads.init`, сессия 2026-06-09) согласие на персонализированную рекламу выставлено
**безусловно**: `MobileAds.setUserConsent(true)`. Для Google Play (Google требует диалог согласия
на сбор данных для рекламы в приложениях с рекламой для пользователей из ЕЭЗ/Великобритании; для РФ
— формально не обязателен, но Google Play Console может требовать **наличие ссылки на политику
конфиденциальности с описанием использования рекламных данных** независимо от региона).
Минимально необходимое — пункт в политике конфиденциальности (см. отдельную задачу по лендингу).
Если потребуется явный диалог согласия — это отдельная Android-задача (UI consent dialog перед
`MobileAds.initialize`).
