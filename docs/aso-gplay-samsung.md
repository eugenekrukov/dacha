> **платная подписка «Дачник Про», 7 дней триал, БЕЗ рекламы**. Прежняя ad-supported модель

**Пакет (applicationId)**: `ru.dachakalend.app` — флейвор `gplay`, **без суффикса**
(`PAYMENTS_ENABLED=true`, `ADS_ENABLED=false`).

> ⚠️ **Samsung Galaxy Store снят с публикации.** Флейвор `samsung` в коде ещё существует (ad-supported),
> но не публикуется. Этот документ — только про **Google Play**.

## Название (лимит Google Play — 30 символов)

```
Календарь дачника: сад, огород
```

**30 символов** — влезает в лимит Play. (Вариант `…: сад и огород` = 31, не проходит: « и »
между пробелами стоит 3 символа.) Запасные: `Календарь дачника: огород` (25),
`Дачник: сад, огород, посадки` (28).

## Краткое описание (лимит 80)

```
Напоминания о поливе, посадках и уходе за растениями. Погода и урожай.
```

(69 символов. Версия RuStore — **без** слова «бесплатно»: приложение платное по подписке.)

## Полное описание (лимит 4000)

Идентично RuStore — взять из `docs/aso-rustore.md` (раздел «Полное описание»). **Не** использовать
прежний gplay-текст с маркерами «бесплатно» и «поддерживается показом рекламы» — реклама отменена.

## Чувствительные разрешения

```
ACCESS_COARSE_LOCATION / ACCESS_FINE_LOCATION — определение населённого пункта участка для прогноза
  погоды и агро-рекомендаций.

```

> `AD_ID` (`com.google.android.gms.permission.AD_ID`) **больше не декларируется** — Yandex Mobile Ads
> SDK выпилен из gplay/rustore сборок вместе с рекламной моделью.

## Google Play Data Safety — категории для заполнения

Всё собирается, **никому не передаётся** (третьим лицам — «Нет»):

| Категория данных | Собирается? | Передаётся? | Назначение |
|---|---|---|---|
| Местоположение (приблизит./точное) | Да | Нет | Функциональность (погода, климат-зона) |
| Email-адрес | Да | Нет | Аккаунт / вход |
| ID пользователя | Да | Нет | Функциональность |
| Контент приложения (посадки, действия, урожай) | Да | Нет | Функциональность |
| ID устройства (push-токен) | Да | Нет | Уведомления |
| История покупок (статус подписки) | Да | Нет | Функциональность (платный доступ) |

> ⚠️ **Главное отличие от прежней карточки**: строка «Рекламный ID (Advertising ID), передаётся
> рекламной сети Яндекса» **УДАЛЕНА** — рекламы больше нет, ничего третьим лицам не передаётся.
> «История покупок (статус подписки)» — теперь **применима** (биллинг включён, `PAYMENTS_ENABLED=true`).

## App access (доступ для проверки Google)

Функции за логином/подпиской → в разделе **App access** дать ревьюеру **тестовый аккаунт**
(email + пароль), иначе ревью отклонят. Завести **отдельный** review-аккаунт (не использовать
основной тест-логин разработчика).

## Релиз — закрытое тестирование

Личный аккаунт разработчика (после 13.11.2023) → перед production обязательно **closed testing**:
≥12 тестировщиков, непрерывно ≥14 дней. Подробности и SHA-1 после первой загрузки —
`docs/gplay-publishing-guide.md`.
