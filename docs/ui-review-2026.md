## 1. Вердикт

UI **зрелый и аккуратный**, с цельным узнаваемым стилем — не выглядит «шаблонным». Хорошая
типографическая иерархия, дружелюбный тёплый бренд, консистентная карточная система. Это уверенный
уровень «выше среднего по категории» (большинство РФ-конкурентов — текстовые базы советов).

Доработки нужны **точечные**, не редизайн. Два главных рычага: **(1) изображения растений**
и **(2) контраст текста на оранжевых поверхностях**. Остальное — полировка.

## 2. Что сделано хорошо (сохранить)

- **Цельный бренд «Sunny Market»**: оранжево-янтарный hero + кремовый фон (`#FFF8EB`), шрифт
  Nunito (округлый, тёплый), крупные жирные начертания. На 100% попадает в аудиторию «дача/40+».
- **Иерархия экрана «Сегодня»**: hero-погода → прогноз → задачи дня. Считывается мгновенно.
- **Крупная типографика** — хорошо для 40+. Есть тумблер «Крупный шрифт» (×1.18 только fontScale,
  вёрстка не ломается) — правильное решение.
- **Семантический цвет задач**: полив — синий, заморозки — красный, урожай — зелёный. Помогает
  быстро различать тип.
- **Нижняя навигация с иконками И подписями**, выделенное активное состояние — для возрастной
  аудитории подписи обязательны (есть).
- **Календарь** с точками-маркерами задач по дням — понятно и опрятно.
- **Карточка культуры** с сегмент-табами (Уход/Болезни/Вредители/Соседи) — грамотная архитектура.
- Консистентные радиусы, отступы, воздух между блоками.

## 3. Находки и рекомендации (по приоритету)

### 🔴 Высокий приоритет

**UI-1. Нет изображений растений нигде.** Карточки культур, посадок и справочника — только текст.
Для приложения про растения и для аудитории 40+ это главный визуальный пробел: фото ускоряет
поиск, повышает доверие и конверсию, и это то, чем выигрывают конкуренты (PictureThis, Planta).
→ Добавить фото топ-30 культур (бэклог **F4**, поле `crops.image_url`). Эскиз: миниатюра слева в
карточке списка + крупное фото в шапке детали.

**UI-2. Контраст текста на оранжевом hero ниже нормы.** Белый текст на `#FF7B00` ≈ **2.6:1**
(WCAG AA требует 4.5:1 для обычного и 3:1 для крупного — не проходит даже крупный). Подписи
«СЕГОДНЯ», «Новосибирск», метео-строка плохо читаются на ярком солнце (а это уличный сценарий!).
→ Варианты: затемнить hero-градиент (старт темнее `#E66A00`/`#D45F00`), либо тёмно-коричневый текст
на светлой части градиента, либо лёгкий тёмный scrim под текстом. Проверить все белые-на-оранжевом
поверхности (CTA-кнопки, чипы) автоматически на контраст.

**UI-3. Нет поиска/фильтра по культурам.** В «Справочнике культур» только чипы-категории; список
длинный, листать утомительно. → Строка поиска + (бэклог **F5/П7**). Дешёвая, заметная победа.

### 🟠 Средний приоритет

**UI-4. Прогноз обрезан у правого края** (карточка «СР» наполовину за экраном). Аффорданс
горизонтального скролла слабый. → Добавить «подглядывание» следующей карточки + градиент-затухание
у края или стрелки; так пользователь поймёт, что строка прокручивается.

**UI-5. Метео-строка в hero мелкая и низкоконтрастная** («Ясно 16…30° · 0%» — мелкий текст на
оранжевом). → Укрупнить и/или вынести на более контрастную подложку (см. UI-2).

**UI-6. Пустые / загрузочные / ошибочные состояния.** В роадмапе отмечены как требующие доработки.
Особенно важны экраны ошибки сети (см. инсайт «офлайн» в research §4) — должны быть дружелюбными
и подсказывать действие, а не пустой экран.

### 🟢 Низкий приоритет / полировка

**UI-7. Только светлая тема** (`lightColorScheme`, тёмной нет). Для 40+ некритично, но тёмная тема —
современное ожидание; как минимум проверить «высококонтрастный» режим для улицы.

**UI-8. Android: эмодзи на вторичных экранах** (web уже на lucide-иконках). Привести к единому стилю
иконок — консистентность (хвост этапа 1 в роадмапе).

**UI-9. Аналитика**: убрать сомнительную метрику «дней подряд», добавить простые графики урожая
(уже в роадмапе) — визуально оживит раздел.

## 4. Чек-лист доступности (40+) — что проверить системно
- [ ] Контраст ВСЕХ текстов ≥ 4.5:1 (особенно белое-на-оранжевом: hero, кнопки, чипы) — **UI-2**.
- [ ] Тач-таргеты ≥ 48×48 dp (нижняя навигация — ок; перепроверить чипы категорий ~36 dp).
- [ ] Поддержка системного увеличения шрифта без обрезки (есть тумблер ×1.18 — проверить экстрим).
- [ ] Альтернативный текст для будущих изображений (UI-1).
- [ ] Фокус/состояния нажатия видимы.

## 5b. Android-приложение — структурный ревью (по Compose-исходникам)

> Метод: чтение `TodayScreen.kt`, `CropsScreen.kt`, `Theme.kt`/`Type.kt` (без устройства).
> Вывод заметно отличается от веб-ревью: **Android-клиент полированнее, чем веб.**

### Что в приложении уже сделано лучше, чем в вебе
- **Hero «Сегодня на даче»** (`SunnyHero`): диагональный срез (`DiagonalBottomShape`), оранжевый
  градиент, polka-dot-оверлей, анимированный подсолнух 🌻. Hero вынесен из `LazyColumn`, чтобы clip
  не срезался при скролле — аккуратное решение.
- **UI-2 (контраст) уже частично закрыт:** на белый текст hero наложены **тени** (`Shadow`) с
  прямым комментарием разработчика про a11y для «аудитории 50+». В вебе теней нет — поэтому веб
  выглядел хуже. ⚠️ Формально по WCAG тень не засчитывается: белый на `#FF7B00` ≈ 2.6:1, мелкая
  дата 13sp всё ещё ниже AA. Для строгого соответствия — затемнить `HeroGradientStart` (напр.
  `#D45F00`). Но в реальной читаемости тени + крупный жирный шрифт проблему сильно смягчают.
- **UI-3 (поиск) в приложении ЕСТЬ:** `CropsScreen` имеет `OutlinedTextField` «Поиск культуры…»
  + чипы категорий. Это веб-пробел, не общий.
- **UI-4 (прогноз) решён лучше:** 7-дневный прогноз свёрнут по умолчанию, разворачивается тапом
  по строке с шевроном, внутри — `horizontalScroll`. Никакой «обрезки», как в вебе.
- **UI-6 (состояния) есть:** loading (`CircularProgressIndicator`), error (с кнопкой «Повторить»),
  empty (иконка + текст) — на `CropsScreen` и `TodayScreen` (`LoadingScreen`/`ErrorScreen`).
- **Умные агро-подсказки в погоде:** «почва холодная — посев не рекомендуется», «ожидается дождь —
  задачи полива скрыты». Это сильный продуктовый штрих, которого не видно в вебе.
- **Жесты:** свайп по задаче (отложить/удалить), онбординг-коачмарки.

### Что подтверждается как реальный пробел приложения
- **🔴 UI-1 (нет фото растений) — да, актуально и в приложении.** `CropCard` использует иконку
  `Icons.Default.Eco`, а не фото. Главный кросс-платформенный пробел. → бэклог **F4**.
- **🟠 Эмодзи на вторичных экранах** (`❄️ Боится заморозков`, `🌿 Многолетник`, `🌧/🌱/⚠️` в погоде) —
  смешаны с lucide-стилем иконок. Консистентность (уже в роадмапе, хвост этапа 1).
- **🟢 Тач-таргеты hero-иконок 44 dp** (`IconButton(Modifier.size(44.dp))`) — чуть ниже
  рекомендованных 48 dp. Мелочь, но для 40+ лучше поднять.

### Переоценка: главный кандидат на доработку — это ВЕБ
Большинство пунктов §3 (поиск, прогноз, состояния, тени в hero) в приложении уже решены. Значит
правильная задача — **подтянуть веб до уровня приложения** (портировать поиск, сворачиваемый прогноз,
тени/контраст hero, состояния), а не «чинить приложение».

## 5c. Визуальная сверка приложения (по реальным скриншотам, верифицировано)

> Осмотрены живые кадры Android-сборки: «Сегодня» (задачи + советы), Календарь, Посадки (+ меню,
> «О посадке», «Уход»), Журнал действий, лист «Записать действие», Справочник проблем + деталь,
> карточка культуры (Болезни/Уход), «Информация», Настройки (3 экрана). Подтверждает выводы §5b.

### Подтверждённые сильные стороны (выглядит дорого для категории)
- **Hero «Сегодня на даче»** вживую premium: оранжевый градиент с диагональным срезом, анимированный
  подсолнух 🌻, белый заголовок с тенью, чипы «💧 28%». Тени реально вытягивают читаемость.
- **Дизайн задач:** плитки с семантичным цветом (зелёная «уход», синяя «полив»), бейдж просрочки
  **«+3д»** красным, «Сделайте сегодня». Считывается мгновенно.
- **«Советы дня»:** карточки культур + отдельная **лунная** карточка (фиол. фон, 🌙 «Растущая луна…»)
  — приятный акцент, «Показать ещё (3)».
- **Погода умная:** «🌧 Дождь 0%», «🌱 Почва 31°», сворачиваемый «Прогноз на 7 дней».
- **Справочник проблем — С ФОТО** (деталь «Недостаток калия»: реальное фото + кредит Wikimedia),
  теги, блок «Где встречается» (Огурец/Томат) — отличная перелинковка. Значит фото-инфраструктура есть.
- **Настройки** полные и сгруппированы: Аккаунт (пароль/email/удаление), Подписка («Дачник Про»,
  «Продлить» — ручное продление как задумано), Уведомления (тумблеры с порогами ≤2°/≥35°), Данные,
  О приложении (версия, оферта, политика, поддержка), «Выйти». Store-compliant.

### Подтверждённые доработки (по приоритету)
- **🔴 UI-1. Фото — только в справочнике проблем, НЕ в культурах/посадках/задачах.** Карточки культур
  и плитки задач используют иконки/глиф листа, не фото растения. Фото в guide доказывает, что движок
  готов → распространить на топ-культуры (**F4**). Главный визуальный апгрейд.
- **🟠 UI-2. Контраст hero подтверждён как мягкая проблема.** Крупный заголовок с тенью читается ок;
  слабые места — мелкая дата «СУББОТА · 20 ИЮНЬ» и текст чипа «💧 28%» (белое на оранжевом ~2.6:1).
  Точечно: усилить фон чипа / чуть затемнить старт градиента.
- **🟠 UI-10. Смешанный язык иконок.** Рядом живут кастомные/lucide-иконки и эмодзи: 🌻 (hero — ок,
  намеренно), но 🌧🌱⚠️❄️🌿 в карточках, 🦠 у болезней, 🍂 в заголовках, 🌙 у луны, ✅ в «Боится
  заморозков». Стоит решить: эмодзи как осознанный стиль ИЛИ унифицировать в один icon-set
  (перекликается с «эмодзи→иконки» в роадмапе).
- **🟢 UI-11. Обрезка первого таба** в карточке («…садке» = «О посадке»). Таб-ряд скроллится, но
  первый ярлык подрезан — лучше делать видимым целиком или добавить индикатор скролла.
- **🟢 UI-12. «Серия дней» (streak)** всё ещё в «Статистике» — роадмап предлагал убрать как
  сомнительную метрику. Подтвердить решение.
- **🟢 UI-13. Версия в «О приложении» — 1.0.1**, а в `build.gradle.kts` versionName `1.0.2`. Вероятно,
  скриншот со старой сборки; свериться при следующем релизе.
- **🟢 UI-9 (тач-таргеты hero-иконок 44 dp)** — без изменений, чуть ниже 48 dp.

### Итоговый вердикт по приложению
UI приложения **полированный и выше среднего по категории** — hero, дизайн задач, умная погода,
лунные советы, полные настройки. Нужен **не редизайн, а лёгкая полировка**. Единственный крупный
визуальный рычаг — **фото культур (F4)**; остальное косметика (контраст hero, единый язык иконок,
обрезка таба). Слабое звено платформы — **веб**, его надо подтянуть к приложению.

## 5. Связь с продуктовым бэклогом
- **UI-1 → F4** (фото культур), **UI-3 → F5/П7** (поиск) — уже в плане, UI-ревью повышает их приоритет.
- **UI-2 (контраст)** — НОВОЕ, дёшево, влияет на доступность и уличную читаемость. Сделать вместе с F4.
- **UI-6 (состояния ошибок)** связано с инсайтом «офлайн» (**F1**) — проектировать заодно.

**Рекомендованный UI-спринт:** UI-2 (контраст, дёшево) + UI-3 (поиск, дёшево) + UI-4 (прогноз) →
затем UI-1 (фото, среднее) как отдельная фича с заметным визуальным эффектом.
