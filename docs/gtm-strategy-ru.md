# GTM-стратегия «Календарь дачника» — рынок РФ (2026)

> Роли: интернет-маркетолог + продюсер. Акцент по запросу — **максимальная автоматизация**.
> Опора: исследование [product-research-2026.md](product-research-2026.md), ASO [aso-rustore.md](aso-rustore.md),
> promo-план [rustore-promo.md](rustore-promo.md). Контекст: free+premium («Дачник Про»),
> без рекламы; каналы дистрибуции — RuStore, Google Play, Web.
>
> **⚠️ Смена модели монетизации (2026-07-18):** 7-дневный триал отменён — не успевал сформировать
> привычку до paywall. Теперь 1 сад и до 3 посадок бесплатно навсегда, «Дачник Про» снимает лимит.
> Документ ниже писался под старую триал-модель — метрики и email-цепочка (§5.2, §6) упоминают
> «конверсию триал→оплата» и нуждаются в отдельном пересмотре под free→paid; воронка §3 тоже.

## 1. Позиционирование

**Для кого:** дачник РФ 40+, у кого участок и нет времени держать в голове весь сезон.
**Что:** личный помощник по саду и огороду — сам подсказывает, что делать сегодня, с учётом культур,
погоды и климата участка.
**Чем отличаемся:** заточен под РФ (культуры/сроки/зоны/лунный календарь), **без рекламы**,
управление участком + дневник урожая. Не «база советов» и не зарубежный трекер комнатных.

**Оффер-крючок:** «Не помни весь сезон — приложение напомнит само». 7 дней бесплатно, без рекламы.

## 3. Главный ростовой цикл — «сезонный контент-двигатель» (P0)

Самый частотный запрос аудитории — **«лунный посевной календарь»** и **«когда сажать X»**.
У приложения уже есть лунные рекомендации и сроки по культурам/зонам → это контент-актив.

**Петля:**
```
данные приложения (лунный календарь, сроки по культурам/региону)
      ↓ авто-генерация
SEO-страницы/посты «Лунный календарь на <месяц>», «Когда сажать <культура> в <регион>»
      ↓ органический трафик
лендинг/веб-приложение → установка → 7-дневный триал
      ↓ email + push lifecycle
оплата «Дачник Про» → реферальный промокод → новый трафик
```

**Автоматизация:** ежемесячный/еженедельный генератор контента (см. §5.1), публикующий в блог
лендинга, Telegram, VK, Дзен из одного источника. Прогон текста через `humanizer-ru` перед публикацией.

## 4. Воронка и метрики (что автоматически мерить)

| Этап | Метрика | Цель-ориентир |
|---|---|---|
| Awareness | органические показы/визиты лендинга | рост MoM |
| Acquisition | установки по каналам, CR карточки стора | ASO-итерации |
| Activation | % дошедших до «участок + 1 посадка», D1 retention | >40% активация |
| Revenue | триал→оплата, день оплаты в триале | >10% триал→оплата |
| Retention | D7/D30, открытия по пушу | сезонная кривая |
| Referral | погашенные промокоды на 1 платящего | >0.3 |

⚠️ Исключать тест-аккаунты (`test@`, `demo@`, `deploytest_`) из метрик.
Дашборд — автогенерируемый отчёт (см. §5.4).

## 5. План автоматизации (ядро запроса)

> Инфраструктура под автоматизацию уже есть: Node-бэкенд, отправка email, push (FCM/RuStore),
> cron (есть `nalogJob`). Ниже — что добавить.

### 5.1 Контент-машина (P0)
- **Источник:** эндпоинт-выгрузка лунного календаря и сроков по культурам/регионам из БД.
- **Генератор:** скрипт (cron, еженедельно) формирует пост/страницу по шаблону → `humanizer-ru` →
  черновик на ревью → публикация.
- **Дистрибуция из одного источника:** блог лендинга (SEO) + Telegram-канал (бот-постинг) +
  VK (API) + Дзен. Кросс-постинг автоматический.
- **Реализация в Claude Code:** запланировать через `/schedule` (cron-агент) еженедельную генерацию
  контент-пакета; ручная только финальная вычитка.

### 5.2 Email lifecycle триала (P0)
Автоцепочка по дню триала (у бэкенда есть `KEY_FIRST_LAUNCH`/`subscription_until` и отправка писем):
| День | Письмо | Цель |
|---|---|---|
| 0 | Добро пожаловать + «создай участок» | активация |
| 1 | «Вот что приложение подсказывает на этой неделе» | value |
| 3 | Кейс/фича (фото-диагностика/справочник) | вовлечение |
| 5 | «Твой сезон в цифрах»: посадок N, задач M | ценность перед оплатой |
| 6 | Напоминание: триал кончается завтра + оффер | конверсия |
| 8 | Win-back, если не оплатил: промокод-скидка | возврат |

Связать с **U7** (ценностный paywall на дне 5–6) и запросом оценки RuStore (день 6) — единый момент.

### 5.3 Реферальная петля (P1)
- Авто-генерация персонального промокода каждому платящему (таблица `promo_codes` уже есть).
- «Подари другу неделю Про» → погашение трекается → дашборд. Полностью авто.

### 5.4 Авто-отчётность (P0, дёшево)
- Еженедельный отчёт метрик воронки (§4) из БД → email/Telegram продюсеру.
- Уже есть локальные скиллы `/statistic`, `/statistic_user` — расширить до сводки по каналам/конверсии
  и поставить на `/schedule` (cron-агент Claude Code).

### 5.5 ASO/фичеринг-автоматизация (P1)
- Сезонные **события RuStore** по календарю (см. rustore-promo.md) — напоминание-крон на создание.
- Мониторинг отзывов/рейтинга → авто-алерт при падении (вход в фичеринг зависит от рейтинга).

### 5.6 Push-реактивация (P0, инфраструктура есть)
- Триггерные пуши: заморозки/жара (есть), «пора сеять <культура>» по региону, «давно не заходил».
- Сегментные кампании по этапу воронки.

## 6. Бюджет/ресурсы и приоритеты соло-режима

Под соло-самозанятого, без рекламного бюджета на старте:
1. **P0 органика + автоматизация** (контент-машина, ASO, email/push lifecycle, отчётность) — почти 0 ₽,
   максимум рычага. **Делать первым.**
2. **P1 коммьюнити + реферал** (Telegram/VK/Дзен, промокоды) — время, не деньги.
3. **P2 платный перфоманс + партнёрства** — включать после подтверждения триал→оплата >10%
   и положительного unit-экономики (LTV > CAC).

## 7. Дорожная карта запуска (90 дней)

**Месяц 1 — фундамент автоматизации**
- Email lifecycle триала (§5.2) + U7 ценностный paywall.
- Авто-отчёт воронки (§5.4) на `/schedule`.
- Запустить Telegram-канал + VK-сообщество (шаблоны постов).
- Первое сезонное событие RuStore.

**Месяц 2 — контент-двигатель**
- Контент-машина (§5.1): лунный календарь + «когда сажать» → блог/Telegram/VK/Дзен.
- Реферальные промокоды (§5.3).
- ASO-итерация по реальным запросам из консолей.

**Месяц 3 — масштаб**
- Триггерные push-кампании по сезону (§5.6).
- Флагманский маркетинговый повод: запуск фото-диагностики (F2) → PR-волна, фичеринг-заявка.
- Тест платного перфоманса на узкой аудитории, если unit-экономика сходится.

## 8. Зависимости от продукта (из research §5)
GTM усиливается этими фичами — синхронизировать приоритеты:
- **U7** ценностный paywall — напрямую под §5.2 (конверсия триала).
- **F1 офлайн** — снижает отток (retention в §4).
- **F2/F3 фото-AI** — главный маркетинговый повод месяца 3 и фичеринга.
- **Реферальные промокоды** — переиспользуют существующую `promo_codes`.
